Overview & Problem

The Challenge

Biomedical researchers working on target ideation and drug discovery face fundamental barriers when attempting to query complex datasets. Fragmented data schemas, specialized query languages like Cypher, and limited semantic interoperability create a technical moat that prevents domain experts from efficiently extracting insights. Research teams—particularly those without dedicated bioinformatics support—spend significant time learning database structures and crafting queries rather than focusing on scientific hypothesis generation.

The challenge becomes even more acute when working with multiple interconnected datasets. Hetionet, a comprehensive biomedical knowledge graph, contains millions of relationships across 47 node types representing genes, diseases, compounds, pathways, and more. Qiagen's IPA database adds nearly 400,000 curated findings from neuropsychiatric and neurodegenerative research. Researchers need to traverse these knowledge graphs to identify novel therapeutic targets, understand disease mechanisms, and discover drug repurposing opportunities—but doing so requires both technical expertise and a priori knowledge of data structures.

The Solution

We developed an AI-augmented target ideation chatbot that enables natural language access to interconnected biomedical knowledge graphs. The system implements an agentic RAG framework with 20+ modular retrieval tools, allowing researchers to ask complex questions without deep technical expertise. By orchestrating multiple specialized agents through LangGraph, the platform intelligently selects appropriate retrieval strategies, expands searches when initial results are insufficient, and synthesizes findings from both Hetionet and Qiagen databases.

Impact

The chatbot democratizes access to complex biomedical data, enabling New Target Team researchers to explore hypotheses interactively without writing Cypher queries or understanding graph schemas. The system supports both simple entity lookups ("What genes are associated with schizophrenia?") and complex multi-hop path reasoning ("Find compounds that target proteins upregulated in Alzheimer's disease and are already FDA-approved for other indications"). This accelerates the target ideation process and allows researchers to iterate on scientific questions in real-time.

Technical Architecture

Technology Stack

  • Knowledge Graphs: Neo4j hosting Hetionet and Qiagen IPA databases with semantic embedding layer
  • Agent Orchestration: LangGraph StateGraph for managing tool selection and retrieval workflows
  • Large Language Models: GPT-4 for complex reasoning and synthesis, Mistral Large for simpler classification and extraction tasks
  • Embeddings: Multiple biomedical embedding models evaluated through LLM-as-judge pipeline (PubMedBERT, BioBERT, BioLinkBERT, SciBERT, BiolinicalBERT, and others)
  • Backend Platform: Dataiku with custom Python libraries for data processing and tool integration
  • Query Language: Cypher for graph traversal, with text-to-Cypher translation layer
  • Frontend: OpenAI chatbot interface for natural language interaction

Implementation & Approach

Modular Tool Architecture (20+ Tools)

The system implements a modular retrieval architecture where each tool specializes in a specific type of biomedical query. Tools are organized into five main categories, with multiple sub-tools within each category optimized for different data sources and query patterns:

1. Keyword Retrieval

Direct entity lookup using exact matches across both knowledge graphs. Ideal for simple queries requiring fast access to known entities, compounds, or diseases.

2. Text-to-Cypher Translation

LLM-powered conversion of natural language questions into executable Cypher queries. Includes query validation, testing, and error correction loops to ensure syntactic correctness before execution.

3. Semantic Retrieval (Hetionet)

Embedding-based search over Hetionet graph with topic-level filtering. Supports concept-level queries and relationship discovery through vector similarity search with additional filtering by node type.

4. Semantic Retrieval (Qiagen)

Semantic search over ~400,000 curated findings from neuropsychiatric and neurodegenerative research. Enables researchers to find relevant evidence from literature without knowing specific entity names.

5. Graph Traversal & Path Ranking

Multi-hop path discovery in Hetionet using Neo4j's shortest path algorithms. Paths are ranked by genetic evidence strength and biological plausibility. Currently implemented for Hetionet; pathway retrieval for Qiagen data is under consideration.

Agentic RAG with Intelligent Expansion

The system goes beyond traditional RAG by implementing an evaluation-and-expansion loop. After initial retrieval, a large language model evaluates whether the retrieved data sufficiently addresses the user's question. If the initial results are insufficient or miss critical context, the agent autonomously selects additional retrieval strategies to expand the search:

  • Initial Retrieval: Small LLM (Llama 3.1-8b) extracts keywords and topics from the user's question to guide initial tool selection
  • Evaluation Step: Large LLM-in-the-loop assesses whether retrieved data addresses the question and determines next actions
  • Expanded Search: If needed, agent triggers additional tools (Hetionet semantic retrieval, graph traversal, or Qiagen semantic search) to enrich context
  • Synthesis: Final LLM call synthesizes findings, explains discovered paths, and provides citations back to the original data sources

Embedding Model Evaluation Pipeline

To optimize semantic retrieval quality, we developed an LLM-as-judge pipeline for systematically evaluating embedding models in the biomedical domain. Using 36 schizophrenia-related target ideation questions as test queries, we:

  • Generated embeddings for questions and findings using multiple biomedical models (PubMedBERT, BioBERT, BioLinkBERT, SciBERT, BioClinicalBERT, ModernBERT, RoBERTa, SapBERT, Snowflake-Arctic)
  • Retrieved top-ranked findings from Qiagen database for each embedding model via cosine similarity
  • Prompted GPT-4 with predefined evaluation criteria to score the semantic relevance of retrieved findings
  • Ranked embedding models based on GPT-4 evaluations and selected top performers (BioLinkBERT for gene-drug associations, Snowflake-Arctic for balanced biological context)

This evaluation framework is generalizable to other research questions and datasets, allowing continuous refinement of retrieval quality as new embedding models emerge.

Design for Multi-Source Integration

A key architectural decision was designing tools to be nearly agnostic of specific graph schemas while currently supporting Hetionet. Each tool operates on abstracted node and relationship types, with schema-specific logic encapsulated in configuration layers. This design enables future expansion to additional biomedical databases (e.g., DrugBank, KEGG) by adding new configuration mappings rather than rewriting core retrieval logic.

The Qiagen integration currently supports keyword and semantic retrieval but not pathway analysis, as the findings database is structured differently from Hetionet's relationship-rich graph. As we refactor Qiagen data to align with graph-based schemas, pathway tools can be extended to this data source.

Technical Challenges & Solutions

Challenge: Query Validation

Text-to-Cypher translation is error-prone due to schema complexity and LLM hallucination. We implemented a query validation loop where generated Cypher is tested against a development database before execution. If the query fails, error messages are fed back to the LLM for correction, with a maximum of three retry attempts.

Challenge: Balancing LLM Cost vs. Quality

GPT-4 provides superior reasoning but is expensive for high-frequency operations. We strategically use Mistral Large for keyword extraction and simple classification tasks, reserving GPT-4 for complex synthesis, evaluation, and Cypher generation. This reduces API costs while maintaining output quality.

Challenge: Semantic Search Relevance

Initial embedding-based retrieval returned too many low-relevance results. We added topic-level filtering for Hetionet (allowing users to specify entity types like "genes" or "compounds") and implemented query expansion techniques where the LLM generates related search terms before retrieval.

Technical Highlights

What Makes This Different

  • Dual Knowledge Graph Integration: Most biomedical chatbots query a single database. We integrate Hetionet's relationship-rich graph with Qiagen's curated findings, enabling cross-source evidence synthesis.
  • Modular Tool Architecture: Rather than a monolithic RAG system, we decompose retrieval into 20+ specialized tools that can be composed dynamically based on question complexity.
  • Intelligent Search Expansion: The system doesn't just retrieve and return—it evaluates, decides, and expands search strategies autonomously when initial results are insufficient.
  • Embedding Model Rigor: We don't assume off-the-shelf embeddings work well for biomedical queries. Our LLM-as-judge evaluation pipeline systematically tests and selects optimal models for this domain.
  • Production-Grade Design: Built on Dataiku's enterprise platform with proper error handling, query validation, and retry logic—designed for organizational deployment rather than prototype demonstrations.

Key Learnings

  • LLM Selection Matters: GPT-4's superior reasoning was essential for Cypher generation and synthesis tasks, while smaller models sufficed for keyword extraction—highlighting the importance of task-specific LLM selection.
  • Graph Schema Complexity: Hetionet's 47 node types and hundreds of relationship types make schema-free querying nearly impossible. Text-to-Cypher works best when constrained to common query patterns rather than attempting full schema coverage.
  • Embedding Quality Varies: Domain-specific biomedical embeddings (BioLinkBERT, BioBERT) significantly outperformed general-purpose models on gene-disease association tasks, validating the investment in embedding model evaluation.
  • User Expectations: Researchers expect chatbots to "know" the data structure implicitly. Designing tools to abstract away schema complexity while providing explanatory paths was critical for usability.
